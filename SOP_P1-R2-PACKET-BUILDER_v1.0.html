Role Purpose & Context

Why this role exists: High-cognition researchers should focus on market intelligence and strategic calls, not formatting. This role offloads all packet structure, visual polish, and data entry work so researchers can move faster through the R1‚ÜíR2‚ÜíR3 pipeline.

In CaniCol v4.0: We need 45-60 R3-ready packets/month to fuel P1 tests. Every hour researchers spend on formatting is an hour not spent finding winners. The Packet Builder role ensures:

  • Speed: Researchers hand off rough dossiers and get back polished packets in 4-8 hours
  • Consistency: Every packet follows the exact same structure, making R3 reviews faster
  • Quality: No broken links, missing screenshots, or ID errors reach the R3 Lead
  • Scalability: As research volume grows, formatting doesn't become a bottleneck
üí° Strategic Value

This role enables researchers to handle 3-4 R3 packets/week instead of 2-3. That 33% throughput gain compounds across the entire P1 testing funnel.

Workflow 2: Screenshot Capture & Visual Assets

Purpose

Capture high-quality, annotated screenshots of competitor products, ads, supplier pages, and pricing to create a visual research record.

Tools

  • Browser extensions: Full Page Screen Capture, GoFullPage, or similar
  • Annotation tool: Skitch, Snagit, or built-in OS tools
  • File naming standard: PRD-YYYYMMDD-XXX_[type]_[source].png

Screenshot Types (Minimum Set)

Screenshot Type Quantity Purpose
Product page (main competitor) 1-2 Show offer structure, pricing, hero image
Competitor ads (FB/TikTok) 3-5 Show creative angles and hooks
Supplier listing 1-2 Show product specs, MOQ, lead time
Price comparison table 1 Show our margin opportunity
Customer reviews 1-2 Show pain points and objections

Step-by-Step

  1. Open all URLs from researcher notes in separate tabs
  2. For each competitor product page:
    • Capture full page screenshot
    • Annotate: circle price, highlight key claims, note shipping
    • Save as: PRD-YYYYMMDD-XXX_competitor_[name].png
    • Upload to Drive: 01_Research_Raw/
  3. For ads (use FB Ads Library or TikTok Creative Center):
    • Screenshot 3-5 top-performing ads
    • Note ad copy hooks in annotation
    • Save as: PRD-YYYYMMDD-XXX_ad_[platform]_NN.png
  4. For supplier pages:
    • Capture product listing + specs
    • Annotate: unit price, MOQ, shipping terms
    • Save as: PRD-YYYYMMDD-XXX_supplier_[name].png
    • Upload to Drive: 03_Supplier_Docs/
  5. Create a price comparison table in Sheets or Notion:
    • Columns: Source, Retail Price, Supplier Cost, Margin %
    • Screenshot this table
    • Embed in Notion packet
  6. Organize all screenshots in Drive folders
  7. Link Drive folder in Notion packet (embed images inline)
‚ö†Ô∏è Common Mistakes

  • Filenames are generic like "Screenshot 1.png" ‚Üí Use structured naming
  • No annotations ‚Üí Add arrows/highlights so R3 Lead can scan quickly

Formatting Standards

  • Headings: Use Notion H2 for main sections, H3 for subsections
  • Tables: Always format tabular data in Notion tables (not text)
  • Images: Embed inline, not as separate blocks at the end
  • Links: Use descriptive text, not raw URLs (e.g., "AliExpress Supplier" not "https://...")
  • Callouts: Use for risk flags and important notes

Step-by-Step Build Process

  1. Duplicate the Notion template "Product Packet Template" from SOP Library
  2. Rename page: PCT-PRD-YYYYMMDD-XXX-R2NN ‚Ä¢ [Product Name]
  3. Fill Meta section: All IDs, links, researcher, date
  4. Copy researcher notes into appropriate sections (don't just paste everything into Overview)
  5. Build tables:
    • Competitor pricing table (Name | Price | URL | Notes)
    • Supplier options table (Supplier | Cost | MOQ | Lead Time | URL)
  6. Embed screenshots from Drive directly into Notion (use /image command)
  7. Format all URLs as named links
  8. Add property tags (in Notion DB):
    • Status: R2_FORMATTING ‚Üí R2_QA ‚Üí R3_READY

    • Date: [today]

Workflow 5: 12-Point QA Checklist

Run this before every R3 handoff. If any item fails, fix it before moving forward.

  • IDs Complete: Product ID and Packet ID present in Notion and ClickUp
  • Drive Folder: Folder structure created, all files uploaded with correct naming

Escalation & Decision Rights

You Can Decide Alone:

  • How to organize Drive folders (within canon structure)
  • Which screenshot tool to use
  • How to annotate screenshots for clarity
  • Order of processing packets (based on due dates)
  • Minor formatting adjustments (fonts, spacing, etc.)

You Must Escalate To Researcher When:

  • Research data is incomplete (missing pricing, no supplier, unclear angle)
  • URLs are broken or don't match product description
  • Screenshots don't show required info (e.g., price is cut off)
  • You can't determine which competitor is "primary"

You Must Escalate To R3 Lead When:

  • Researcher is unresponsive for >4 hours on missing data
  • You're blocked on 3+ packets due to incomplete research
  • You discover a structural issue with packet template
  • WIP is consistently above 8 packets

You Must Escalate To PM When:

  • ClickUp or Notion workspace is broken (can't access, permissions issue)
  • Drive folder permissions are wrong
  • Researchers are consistently handing off incomplete work
  • You need access to a new tool or integration

Response Time Expectations

Issue Type Escalate To Expected Response

Blocked on multiple packets R3 Lead Same day

‚ö†Ô∏è When NOT to Escalate
