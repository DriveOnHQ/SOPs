1. Role Purpose & Context

Why This Role Exists

DriveOn Commerce operates a three-lane store routing strategy for winners that pass P2:

  • Validation Lane: Fast micro/1-page stores for P1/P2 testing
  • Brand Lane: Deep brand/niche stores for premium winners
  • General/SEO Lane (YOUR LANE): Catalog-style general store for Google Shopping, PMax, and organic search

The General/SEO Lane is where DriveOn aggregates P2_PASS and P3 winners into a unified storefront optimized for:

  • Google Shopping feeds and PMax campaigns
  • Organic search traffic through SEO-friendly collections, internal linking, and schema markup
  • Long-term compounding revenue from products that graduate out of paid testing

How This Connects to CaniCol v4.0

DriveOn tests 45-60 products/month in P1/P2. Only 4-6% become true winners (‚âà2-4 products/month). These winners need a home where they can generate revenue without ongoing high-spend paid media.

The general catalog store is that home. Your job is to build the structural foundation‚Äîthe collections, navigation, filters, PDP templates, and UX patterns‚Äîthat make the store:

  • Easy for Google to crawl and rank
  • Easy for shoppers to navigate and convert
  • Easy for the SEO specialist to optimize and scale
  • Compatible with Google Shopping feeds and PMax campaigns
üí° Key Insight: You're NOT building individual product landing pages (that's P3-SB-VALIDATION). You're building the container system‚Äîthe collections, category pages, and navigation structure‚Äîthat holds 20, 50, or 100+ products as DriveOn scales.

4. Core Workflows

Workflow 1: Create New Collection for Product Category

Trigger: 3+ P2_PASS products in same niche/category OR SEO specialist requests new collection for keyword targeting

Inputs

  • Notion Product OS: List of products in category
  • Keyword research from P3-SEO-GOOGLE (target keywords, search volume)
  • Existing collection structure (to avoid duplication/cannibalization)

Steps

Step 1: Coordinate with P3-SEO-GOOGLE
  • Confirm target keywords and collection URL structure
  • Get SEO-optimized collection title, description, meta description
  • Verify this collection won't cannibalize existing pages
Step 2: Create Collection in Shopify Admin
  • Products > Collections > Create collection
  • Set collection type: Manual or Automated (based on product tags/type)
  • Add collection title (keyword-rich, from SEO specialist)
  • Add collection description (SEO-optimized copy, 150-300 words)
  • Set collection image (high-quality, relevant to niche)
  • Configure SEO settings: URL handle, meta title, meta description
Step 3: Add Products to Collection
  • If automated: Set conditions (product type, tags, vendor)
  • If manual: Add products one by one from Product OS list
  • Set product order: Bestselling, Manual, Price (low to high), etc.
  • Verify all products display correctly with images and pricing
Step 4: Configure Collection Page Layout
  • Use theme editor or page builder (PageFly/GemPages) to customize layout
  • Add content modules: hero banner, category description, trust badges
  • Configure filters: price, tags, product type, custom attributes
  • Set up sorting options: Bestselling, Price, Newest
  • Add schema markup (BreadcrumbList, CollectionPage) if not automated
Step 5: Update Navigation
  • Add collection to main navigation menu (if top-level category)
  • Add to mega-menu or dropdown (if sub-category)
  • Update footer links if relevant
  • Test navigation on desktop and mobile
Step 6: Quality Checks
  • Mobile responsive test (all breakpoints)
  • Page speed test (PageSpeed Insights, target LCP < 2.5s)
  • Filter functionality test (all combinations)
  • Shopping feed validation (check Google Merchant Center)
  • Internal linking check (breadcrumbs, related collections)

Outputs

  • Live collection URL: driveon.com/collections/[collection-slug]
  • ClickUp task updated: Status = Done, URL added, QA checklist completed
  • Handoff in #p3-general-seo: Collection live, ready for SEO optimization
  • Notion Product OS updated: Products linked to collection

Quality Bar

  • Collection loads in < 3 seconds on mobile (4G)
  • All products display with correct images, prices, titles
  • Filters work on all devices
  • Breadcrumbs and navigation are correct
  • Meta title < 60 characters, meta description < 160 characters
  • No 404 errors, no broken images
Workflow 2: Update PDP Template for New Product Attributes

Trigger: New product type requires additional fields (e.g., sizing chart, material specifications) OR SEO specialist requests schema/layout changes

Inputs

  • Template change request from P3-SEO-GOOGLE or media buyer
  • Product attribute requirements (from Product OS or research packets)
  • Current PDP template structure

Steps

Step 1: Assess Template Change Scope
  • Determine if this is a global template change or product-type-specific
  • Identify which products will be affected
  • Check for conflicts with existing metafields or custom fields
Step 2: Create/Update Metafields (if needed)
  • Shopify Admin > Settings > Custom data > Products
  • Add new metafield definitions (e.g., product.specs.material, product.specs.dimensions)
  • Set field type (single line text, multi-line, number, file, etc.)
  • Set validation rules if applicable
Step 3: Edit PDP Template
  • Theme editor > Sections > Product template
  • Add new content blocks for attributes (specs table, sizing chart, etc.)
  • Use liquid tags to display metafields: {{ product.metafields.specs.material }}
  • Style new elements to match existing design system
  • Add schema markup for new attributes (Product, Offer, AggregateRating)
Step 4: Test Template Changes
  • Preview template on 3-5 representative products
  • Test on desktop, tablet, mobile
  • Verify metafields display correctly (or gracefully hide if empty)
  • Check page speed impact (compare before/after)
Step 5: Populate Metafields for Existing Products
  • Bulk edit via Shopify Admin or CSV import
  • Coordinate with P1 research or P3-SB-VALIDATION for data
  • QA sample of updated products

Outputs

  • Updated PDP template: Live on all affected products
  • Metafield documentation: Added to Notion/Drive for future reference
  • Handoff notes: Posted in #p3-general-seo with before/after examples
Workflow 3: Implement Navigation/Filter Changes for SEO

Trigger: SEO specialist requests navigation changes based on search console data OR filter optimization for product discovery

Inputs

  • Change request from P3-SEO-GOOGLE with specific keywords/structure
  • Google Search Console data showing high-impression, low-CTR queries
  • Current navigation structure and filter setup

Steps

Step 1: Map New Navigation Structure
  • Review proposed changes with SEO specialist
  • Sketch navigation hierarchy (top-level > sub-categories > filters)
  • Identify URL structure implications (301 redirects needed?)
Step 2: Update Main Navigation
  • Shopify Admin > Online Store > Navigation
  • Edit main menu: add/remove/reorder items
  • Update mega-menu dropdowns (if using custom theme/app)
  • Set up mobile menu behavior
Step 3: Configure Advanced Filters
  • Use Shopify's native filters OR filter app (Search & Discovery, Product Filter & Search)
  • Add filter options: price, product type, tags, custom attributes
  • Set filter display order and grouping
  • Enable URL parameter tracking for filtered pages (for SEO/analytics)
Step 4: Internal Linking Updates
  • Add breadcrumbs to all product/collection pages
  • Update footer links to reflect new navigation
  • Add "related collections" modules to PDPs
  • Implement faceted navigation breadcrumbs for filtered views
Step 5: 301 Redirect Setup (if URLs changed)
  • Identify old URLs that changed
  • Set up 301 redirects in Shopify Admin > Navigation > URL Redirects
  • Test all redirects (use redirect checker tool)
  • Update sitemap (usually auto-updates in Shopify)

Outputs

  • Live navigation changes: Main menu, mobile menu, footer updated
  • Filter system: Active on all collection pages
  • Redirect log: Documented in Notion/Drive
  • Handoff to SEO: Changes live, ready for Search Console monitoring
Workflow 4: Add New P2_PASS Product to General Catalog

Trigger: Product passes P2 testing and is routed to General/SEO lane

Inputs

  • Notion Product OS: Product marked as P2_PASS, Lane = General/SEO
  • Product data: title, description, images, variants, pricing (from validation store or P1 research)
  • Category/collection assignment from SEO specialist

Steps

Step 1: Verify Product Data
  • Check Shopify product page: title, description, images, variants, pricing
  • Verify metafields are populated (specs, features, shipping info)
  • Ensure product status = Active
Step 2: Assign to Collections
  • Add product to 2-4 relevant collections (category, niche, best-sellers, new arrivals)
  • Set product order within collections (manual or auto-sort)
  • Verify product displays correctly in collection views
Step 3: Optimize PDP for Catalog Store
  • Switch product to catalog PDP template (vs validation store template)
  • Add related products module
  • Add reviews/rating widget (if using review app)
  • Add trust badges, shipping info, return policy
Step 4: Shopping Feed Verification
  • Check Google Merchant Center: product should appear in feed within 24 hours
  • Verify product data: title, image, price, availability, GTIN (if applicable)
  • Fix any feed errors flagged by Merchant Center
Step 5: Update Tracking & Handoff
  • Update Notion Product OS: Catalog_Added = True, Collections_Added = [list]
  • Update ClickUp task: Status = Done
  • Post in #p3-general-seo: Product live in catalog, ready for SEO optimization

Outputs

  • Product live in general catalog: Accessible via collections and search
  • Shopping feed updated: Product appears in Google Merchant Center
  • Notion/ClickUp updated: Product status reflects catalog addition

Quality Standards

Technical Quality Bar

  • Page Speed: LCP < 2.5s on mobile (4G), CLS < 0.1
  • Mobile Responsive: All layouts functional on 320px, 768px, 1024px, 1440px breakpoints
  • Browser Compatibility: Chrome, Safari, Firefox, Edge (latest 2 versions)
  • Accessibility: ARIA labels on interactive elements, sufficient color contrast (4.5:1)
  • Feed Validation: 100% of products pass Google Merchant Center validation

SEO Quality Bar (in partnership with P3-SEO-GOOGLE)

  • Meta Tags: Title < 60 chars, description < 160 chars, keyword-optimized
  • URL Structure: Clean, keyword-rich, hyphen-separated (e.g., /collections/sleep-aids)
  • Schema Markup: Product, Offer, BreadcrumbList, AggregateRating on all PDPs
  • Internal Linking: Breadcrumbs on all pages, related collections on PDPs
  • Image Optimization: Alt tags on all images, file names descriptive, < 200KB per image

UX Quality Bar

  • Navigation Clarity: 3-click rule (any product reachable in 3 clicks from homepage)
  • Filter Functionality: Filters work on all devices, maintain URL state, reset properly
  • Collection Layouts: Consistent grid, clear product info (image, title, price, rating)
  • Product Discovery: Related products, recently viewed, "customers also bought" modules
üí° Good vs Bad Examples:
  • GOOD: Collection "Sleep Aids & Relaxation" with 15 products, keyword-optimized description, filters for price/rating/type, fast load time, mobile-friendly
  • BAD: Collection "Products" with 100+ mixed items, no description, slow load time, broken filters on mobile, no breadcrumbs
