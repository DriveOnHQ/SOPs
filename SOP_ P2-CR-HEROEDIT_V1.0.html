Purpose & Context

Why This Role Exists: The CaniCol v4.0 model tests 45-60 products/month at $40-60 P1 budgets. Most products die at P1. Your hero edits must communicate maximum value in minimal time and spend, giving each product its best shot at passing CTR/CPC thresholds.

How You Support DriveOn's Mission

DriveOn Commerce runs a lean-volume testing engine, not a "bet big on one winner" model. Your role is critical because:

  • Volume Matters: Pod 2 must produce 250-400 assets/month. You handle the "hero" tier ‚Äî the highest-quality, primary test creatives (typically 2-3 per product).
  • First Impressions Win: With $40-60 P1 budgets, we get ~2,000 impressions per product. If the first 3 seconds don't hook, CTR tanks and CPC spikes.
  • Testing Rigor: Your edits feed P1 tests that use hard numeric thresholds (CTR ‚â•1.3%, CPC ‚â§$1.20, depth events). Every frame choice impacts these metrics.
  • Economic Discipline: Pod 2 creative time is limited. You focus on hero edits; volume editors handle variants. This division keeps quality high and WIP manageable.

Connection to Holdings & Brand

All DriveOn entities share a systems-first, anti-hype positioning (see L2-08 Brand Platform). Your edits must:

  • Avoid fake urgency, countdown timers, or dark patterns
  • Focus on clarity, benefit, and authenticity
  • Match the product's niche/tone (e.g., wellness vs fitness vs golf)

Core Workflows

Why This Matters: These are your repeatable processes. Master them to maintain 8-12 edits/week without burnout or quality loss.

Workflow 1: Produce P1 Hero Edit (New Product)

TRIGGER: Creative Director assigns brief
Inputs
  • ClickUp Task: "P1 Hero Edit - [Product_ID]" (in "Hero Edits" list, status = "Active")
  • Notion: Product OS page (Product_ID, niche, AOV, target audience)
  • Drive: /03_P2_Creative/Products/[Product_ID]/01_Briefs/P1_Creative_Brief.pdf
  • Drive: /03_P2_Creative/Products/[Product_ID]/02_Raw_Footage_and_UGC/ (all source files)
  • From Copy: Approved scripts/hooks in brief or Notion "Scripts" property
Steps
  1. Load context: Read P1 Creative Brief. Note: hook angle, first 3-second strategy, CTA, format (9:16 vs 16:9 vs 1:1).
  2. Import assets: Download all raw footage/b-roll from Drive /02_Raw_Footage_and_UGC/ into your editing software (CapCut/Premiere).
  3. Structure timeline:
    • 0-3 seconds: Hook (pattern interrupt, problem/benefit)
    • 3-10 seconds: Value prop (what it is, why it matters)
    • 10-20 seconds: Social proof or demo
    • 20-30 seconds: CTA + offer
  4. Edit first pass: Focus on pacing, clarity, and script adherence. Use jump cuts, b-roll overlays, text overlays as specified in brief.
  5. Apply brand guardrails:
    • No fake countdown timers or scarcity
    • No over-hyped language ("miracle," "secret," "life-changing")
    • Clean, modern aesthetic (avoid neon overload, meme clutter)
  6. Export first draft:
    • Format: MP4, H.264, 1080p minimum
    • Naming: CRT-[Product_ID]-P1-HERO-v1.mp4
    • Save to: Drive /03_P2_Creative/Products/[Product_ID]/04_Exports/P1_Tests/
  7. Request review: Update ClickUp task status to "Review" and assign to Creative Director. Post in #p2-edits-hero:
    üé¨ [REVIEW REQUEST] Product: PRD-20251208-042 Edit: CRT-PRD-20251208-042-P1-HERO-v1 Drive: [link] ClickUp: [task link] Notes: First pass on sleep guard hook. Used "tired of waking up" angle per brief.
  8. Iterate on feedback: Creative Director will either approve or request changes. If changes: re-edit, export v2, re-submit. Typical cycle: 1-2 revisions.
  9. Final approval: Once Creative Director approves, update ClickUp status to "Approved" and save final file as CRT-[Product_ID]-P1-HERO-FINAL.mp4
Outputs
  • Drive: Final approved edit in /04_Exports/P1_Tests/
  • ClickUp: Task status = "Done," assigned to Media Buyer
  • Project File: Saved to /03_Project_Files/ for future P2 iterations
Quality Checks
  • First 3 seconds hook visually and align with script
  • Audio levels balanced (voiceover, music, SFX)
  • No watermarks or non-compliant elements
  • CTA is clear and matches brief (e.g., "Shop Now," "Learn More")
  • Export format matches platform spec (9:16 for TikTok/Reels, 16:9 for YouTube, 1:1 for Feed)

Workflow 2: Produce P2 Iteration Variants

TRIGGER: P1 passes + Creative Director requests P2 variants
Inputs
  • P1 Performance Data: From Pod 3 (CTR, CPC, which hooks worked)
  • Creative Director Brief: "Test new hook" OR "iterate on winning creative"
  • Existing P1 Project File: Drive /03_Project_Files/[Product_ID]/P1_HERO_v1.prproj (or .capcut)
Steps
  1. Review P1 data: Check Pod 3 performance summary. Identify which creative had best CTR/CPC and why (first 3s? specific hook? format?).
  2. Load P1 project file: Open existing project from /03_Project_Files/. Duplicate timeline.
  3. Apply P2 strategy:
    • If testing new hook: Replace first 3-10 seconds with new angle (e.g., problem-based ‚Üí benefit-based)
    • If iterating on winner: Keep hook, adjust middle section (add testimonial, change demo angle, test different CTA)
  4. Export P2 variants: Typically 2-4 new versions. Naming: CRT-[Product_ID]-P2-HERO-v1/v2/v3.mp4
  5. Save to Drive: /04_Exports/P2_Tests/
  6. Request review: Post in #p2-edits-hero with comparison to P1 performance
Outputs
  • Drive: 2-4 P2 variant edits in /04_Exports/P2_Tests/
  • ClickUp: P2 task status = "Done"
  • Discord: Handoff message with P1 vs P2 comparison notes

Workflow 3: Daily Handoff to Media Buyers

TRIGGER: End of editing block (4-5 PM daily)
Steps
  1. Collect completed edits: List all edits exported and approved today.
  2. Post handoff in #p2-edits-hero: Use canonical template (see ¬ßHandoffs below).
  3. Update ClickUp: Change task status from "Review" to "Done"; reassign to Media Buyer (P3 role).
  4. Verify Drive links: Ensure all files are in /04_Exports/ and shareable.
Quality Check
  • All edits have correct file naming (CRT-[Product_ID]-P1/P2-HERO-FINAL.mp4)
  • ClickUp tasks updated with Drive links
  • Handoff message includes Product_ID, CRT_ID, and any special notes

Workflow 4: Performance Review & Learning Loop

TRIGGER: Weekly (Friday afternoon or Monday morning)
Purpose

Review what worked and what didn't in last week's P1 tests to improve future edits.

Steps
  1. Pull P1 performance data: Ask Media Buyer or check Pod 3 dashboard for last week's P1 results (CTR, CPC, pass/fail).
  2. Map edits to outcomes: Which of your hero edits had CTR >1.3%? Which failed (<0.8%)? What were the hook patterns?
  3. Identify patterns:
    • Did problem-based hooks outperform benefit-based?
    • Did UGC-style work better than polished b-roll?
    • Did text overlays help or hurt thumb-stop rate?
  4. Document learnings: Update a shared Notion page "Hero Edit Playbook" with 2-3 takeaways.
  5. Share with Creative Director: Brief async message in #p2-edits-hero: "This week: problem hooks +20% CTR vs benefit hooks. Recommend more problem-first for next batch."
Output
  • Notion: Updated "Hero Edit Playbook" with data-backed patterns
  • Next week's edits: Apply learnings to improve CTR/CPC performance

Escalation & Decision Rights

Why This Matters: Knowing when to decide vs escalate prevents bottlenecks and protects you from making decisions outside your authority.

Decisions You Can Make Alone

  • Choice of editing technique (jump cuts, b-roll overlays, transitions) within brief parameters
  • Minor pacing adjustments (e.g., trim 2 seconds from middle section to improve flow)
  • Selection of stock footage or b-roll from approved asset packs
  • Export format details (bitrate, codec) as long as resolution/aspect ratio match brief
  • Requesting clarification from Creative Director on ambiguous brief elements

Decisions Requiring Creative Director Approval

  • Major script changes: Reordering sections, changing hook angle, altering CTA
  • Format changes: Switching from 9:16 to 16:9, or changing duration (15s ‚Üí 30s)
  • Brand guardrail questions: "Is this countdown timer okay?" (Answer: No, but escalate if unsure)
  • Budget/time overruns: If a product requires >4 hours of editing (significantly above average)
  • Asset gaps: If raw footage is insufficient or wrong format, escalate to Asset Sourcer via Creative Director

Decisions Requiring PM / Founder Escalation

  • WIP cap violation: If you're consistently hitting 6+ products and can't keep up with 8-12/week throughput
  • Tool/software needs: Requesting new editing software or plugin purchases
  • Systemic blockers: If Asset Sourcer is consistently late or Creative Director reviews are taking >48 hours
  • Canon conflicts: If a Creative Director request violates brand guardrails (e.g., "add fake timer")

Onboarding Checklist

Why This Matters: A structured 30-day ramp ensures you're productive and autonomous by week 5, not fumbling with tools or missing canon constraints.

Phase 1: Access & Context (Days 1-3)

  • Day 1: Receive all tool access (ClickUp, Notion, Drive, Discord) and confirm login
  • Day 1: Read this SOP (P2-CR-HEROEDIT) in full, take notes on canon constraints
  • Day 1: Review L1-01 CaniCol Model (understand P1/P2/P3 gates and economics)
  • Day 2: Review L2-05 Testing Canon (memorize P1 pass/kill thresholds: CTR, CPC, depth events)
  • Day 2: Review L2-08 Brand Platform & Creative Guardrails (know what's banned: fake urgency, hype, etc.)
  • Day 3: Shadow Creative Director: Sit in on 1-2 brief reviews to see approval process
  • Day 3: Explore Drive structure (/03_P2_Creative/) and ClickUp Hero Edits lists

Phase 2: Supervised Editing (Days 4-10)

  • Day 4-5: First hero edit (with live Creative Director feedback): Choose a simple product (15s, 9:16, clear brief)
  • Day 6-7: Second hero edit (with async Creative Director feedback): Practice handoff template in Discord
  • Day 8-10: Complete 2-3 hero edits independently, submit for review
  • Day 10: Check-in with Creative Director: Review first-pass approval rate, pacing, and any blockers

Phase 3: Independent Production (Days 11-20)

  • Week 3: Aim for 6-8 hero edits (below full throughput but ramping up)
  • Week 3: Practice full workflow: brief intake ‚Üí edit ‚Üí review ‚Üí handoff ‚Üí Drive organization
  • Week 3: Start tracking time in ClickUp on every task
  • End of Week 3: Mid-onboarding review with Creative Director: Are you on track for 8-12/week?

Phase 4: Full Autonomy (Days 21-30)

  • Week 4: Hit 8-12 hero edits/week (full throughput)
  • Week 4: Complete at least one P2 iteration (practice performance-based editing)
  • Week 4: Participate in weekly performance review loop (analyze P1 data, document learnings)
  • Day 30: 30-day review with Creative Director + PM: Assess KPIs, approve for independent work
