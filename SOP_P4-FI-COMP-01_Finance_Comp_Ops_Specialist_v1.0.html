The Finance & Comp Ops Specialist is the financial backbone of DriveOn Commerce, ensuring that every team member is paid accurately, on time, and in accordance with their compensation profile. In a lean-volume dropshipping operation running the CaniCol v4.0 model, precise financial tracking is critical because:

  • Team trust depends on reliability â€” consistent, error-free payouts build the foundation for high-performing distributed teams

Core Workflows

Workflow 1: Pay Run Execution

ğŸ¯
Trigger

Pay period end date reached (biweekly or semi-monthly per schedule)

Inputs

  • ClickUp Time Tracking: All time entries for pay period
  • ClickUp Deliverables: Completed task counts per person
  • Notion Product OS: P3 winner status for bonus calculations
  • Notion People & Roles DB: Comp profiles and rate tables

Step-by-Step Process

1
Define Pay Period

Action: Create new Pay Run record in Notion Pay Runs DB

Fields to populate:

  • Pay_Run_ID: PAY-YYYYMM-NN (e.g., PAY-202512-02)
  • Period_Start: First day of period
  • Period_End: Last day of period
  • Status: "Open"

Quality check: Verify no overlap with previous Pay Run period

2
Export Time Entries from ClickUp

Location: ClickUp â†’ Time Tracking view (all spaces)

Action: Export time entries for pay period date range

Filter by: Period_Start â‰¤ Entry_Date â‰¤ Period_End

Required columns: Person, Task, Hours, Date, Approval_Status

Quality check: All entries must have Approval_Status = "Approved"

3
Generate Hourly Comp Events

For each person with time entries:

  1. Sum total approved hours
  2. Look up hourly rate from People & Roles DB
  3. Create Comp Event in Notion:
Comp_Event_ID: CMP-YYYYMM-[PersonCode]-01 Person: [Person Name] Comp_Type: "Hourly" Units: [Total Hours] Rate: [Hourly Rate] Calc_Amount: Units Ã— Rate Source: "ClickUp_Time" Status: "Planned" Pay_Run: [Current Pay Run ID]
4
Generate Deliverable Comp Events

For each person with deliverable-based comp:

  1. Query ClickUp for completed tasks in pay period
  2. Count deliverables by type (packets, creatives, listings, etc.)
  3. Look up per-deliverable rate from People & Roles DB
  4. Create Comp Event:
Comp_Event_ID: CMP-YYYYMM-[PersonCode]-02 Comp_Type: "Per_Deliverable" Unit_Ref: [Deliverable Type] Units: [Count] Rate: [Per-Unit Rate] Calc_Amount: Units Ã— Rate Source: "ClickUp_Deliverable"
5
Generate Winner Bonus Events (If Applicable)

Check Product OS for P3 winners in period:

  1. Query Notion Product OS: Status = "P3_Winner" AND Winner_Date in pay period
  2. For each winner, identify eligible team members per bonus structure
  3. Calculate bonus: flat amount OR percentage of net_profit
  4. Create Per_Winner Comp Event

Quality check: Winner status must be confirmed in Product OS before bonus creation

6
Add Retainers & Adjustments

Retainers:

  1. Query People & Roles DB for active retainer arrangements
  2. Calculate pro-rata amount for pay period
  3. Create Retainer Comp Event

Adjustments:

  1. Check for pending adjustment requests
  2. Create Adjustment Comp Events with notes explaining reason
7
Calculate Pay Per Person

Action: Sum all Comp Events per person for this Pay Run

Total_Pay = SUM(Calc_Amount) WHERE Pay_Run = [Current Pay Run ID] AND Person = [Person]

Quality checks:

  • Compare to previous pay run (flag variance >30%)
  • Verify no negative amounts (except valid adjustments)
  • Cross-check comp event count per person
8
Request OWNER Approval

Prepare approval package:

  • Pay Run summary with per-person totals
  • Breakdown by comp type
  • Flagged anomalies (if any)
  • Link to Pay Run record in Notion

Post to: #p4-finance-comp (Discord) or DM OWNER

Update Pay Run Status: "Pending_Approval"

9
Execute Payouts (Post-Approval)

Prerequisites: Pay Run Status = "Approved"

For each person:

  1. Execute payment via PayPal/Wise
  2. Record transaction ID
  3. Update all person's Comp Events: Status = "Paid"

Update Pay Run: Status = "Paid", Paid_Date = today

10
Reconcile & Archive

Reconciliation:

  1. Compare payout amounts to calculated totals
  2. Verify all transaction IDs recorded
  3. Flag any discrepancies for resolution

Archive:

  1. Export Pay Run summary to Google Drive
  2. File path: /DriveOn Commerce/Finance/Pay Runs/[YYYY]/PAY-YYYYMM-NN/
  3. Update Pay Run Status: "Closed"

Notification: Post completion to #p4-finance-comp

Outputs

  • Notion: Complete Pay Run record with all Comp Events
  • Payment Platform: Executed payouts with transaction IDs
  • Google Drive: Archived Pay Run documentation
  • Discord: Team notification of completed pay run
