üéØ 2. Role Purpose & Context

This role builds long-term compounding organic traffic for the General/SEO catalog store, reducing paid-only dependency and creating sustainable revenue channels.

Why This Role Exists

In the CaniCol v4.0 model, DriveOn tests 45-60 products monthly at lean budgets ($40-$60 P1, total $80-$100 P1+P2). Winners achieving P2_PASS (4-6% hit rate) are routed into one of three lanes:

  • Validation Lane: Fast MVS/1-page stores for quick P1/P2 tests
  • Brand Lane: Dedicated brand/niche stores for premium winners
  • General/SEO Lane: Multi-product catalog store optimized for organic search and Google Shopping

The P3-SEO-GOOGLE role owns the General/SEO lane, ensuring that validated winners aren't solely dependent on paid ads but also build organic momentum through:

  • Search-optimized collections and product pages
  • Google Shopping feed health and PMax campaign optimization
  • Technical SEO foundations (CWV, schema, internal linking)
  • Long-tail keyword coverage across product catalog

How This Supports CaniCol Economics

CaniCol's lean-volume testing philosophy means most products fail quickly. The General Store serves as a portfolio aggregator where multiple P2_PASS winners co-exist, each contributing organic traffic that compounds over time:

  • Cost Efficiency: Organic traffic has no per-click cost, improving unit economics for proven winners
  • Reduced CAC: Shopping and organic reduce dependency on paid social CPC inflation
  • Portfolio Effect: Even if individual products have modest organic potential, the aggregate store builds domain authority and captures broad search intent
  • Winner Longevity: Products that might exhaust paid audiences can continue generating revenue through search
Canon Alignment: This role follows standard ecommerce practice where SEO is a distinct specialist function, separate from paid media buying. Per L2-03, this ensures focused expertise on organic growth vs. paid campaign execution.

Relationship to Holdings & Brand Architecture

Per L0-02 (Product & Brand Architecture):

  • DriveOn Commerce operates the General/SEO store as a "Dropshipping Engine" ‚Äì a lean catalog model
  • This role doesn't own branded niche stores (that's P3-SB-BRAND's domain)
  • Focus is on functional SEO for a multi-product catalog, not brand storytelling
  • Optimizations serve the portfolio strategy, not individual product obsession

Workflow 1: New Winner SEO Onboarding

Trigger

Media Buyer confirms P2_PASS winner routed to General Store lane

Inputs

  • Product_ID, Test_ID from P3-MB-BUYER
  • Product packet from P1 (AOV, niche, angles)
  • Winning creative from P2 (for content inspiration)
  • Store structure from P3-SB-GENERAL (collection placement)

Steps

  1. Initial Audit (15-30 min)
    • Check if product already in General Store catalog
    • Review P3-SB-GENERAL's collection placement
    • Identify primary + secondary keyword targets using GSC, Ahrefs/Semrush
    • Check competitor PDPs for content gaps
  2. Product Page SEO Setup (30-45 min)
    • Optimize title tag: [Primary Keyword] | [Benefit] | DriveOn
    • Write meta description (155 chars, include CTA + benefit)
    • Add product schema markup (Product, AggregateRating, Offer)
    • Optimize product description (300-500 words, keyword-natural, benefit-focused)
    • Add alt text to all product images (descriptive + keyword-natural)
  3. Internal Linking (15-20 min)
    • Add 3-5 contextual internal links from related products
    • Add links from relevant collection pages
    • Update breadcrumbs if needed
  4. Shopping Feed Integration (20-30 min)
    • Verify product appears in Merchant Center feed
    • Check all required attributes (title, description, price, image, GTIN/MPN)
    • Validate no disapprovals or warnings
    • If multi-variant, ensure proper variant mapping
  5. Quality Check (10 min)
    • Mobile preview: CWV, layout, CTA clarity
    • Desktop preview: full content, images, schema
    • Test URL in GSC: request indexing

Outputs

  • Product page live with SEO optimizations
  • Product in Shopping feed (verified in Merchant Center)
  • Internal links established
  • ClickUp task: SEO-[Product_ID]-ONBOARD moved to SEO_DEPLOYED
  • Notion SEO Projects DB updated with baseline metrics

Workflow 2: Collection Optimization for Organic Growth

Trigger

New collection created by P3-SB-GENERAL, or existing collection identified for SEO uplift

Inputs

  • Collection URL and product count
  • Keyword research data (GSC, competitor analysis)
  • Current organic performance (sessions, conversions)

Steps

  1. Keyword & Intent Research (30-45 min)
    • Identify primary collection keyword (e.g., "best kitchen gadgets")
    • Find 3-5 long-tail variants (e.g., "innovative kitchen tools 2025")
    • Analyze SERP: featured snippets, PAA, image packs, shopping results
  2. On-Page Optimization (45-60 min)
    • Title tag: [Primary Keyword] | Shop [Category] | DriveOn
    • Meta description: 155 chars, benefit-focused, CTA
    • H1: Primary keyword, natural phrasing
    • Collection description: 400-600 words
      • Intro paragraph with primary keyword
      • 2-3 benefit sections with subheadings (H2)
      • Natural keyword integration (no stuffing)
      • Internal links to top products
    • Add collection schema (CollectionPage or ItemList)
  3. Internal Linking Strategy (20-30 min)
    • Add links from homepage/nav (if tier-1 collection)
    • Cross-link from related collections
    • Add links from 5-10 relevant product pages
    • Update sitemap priority if needed
  4. Image & Media Optimization (15-20 min)
    • Hero image: optimized, descriptive file name, alt text
    • Product thumbnails: consistent sizing, lazy load
    • If using banners, ensure mobile responsive
  5. CWV Check (10-15 min)
    • PageSpeed Insights audit (mobile + desktop)
    • Verify LCP < 2.5s, FID < 100ms, CLS < 0.1
    • Fix critical issues before deployment

Outputs

  • Optimized collection page live
  • Schema implemented and tested
  • Internal linking network established
  • GSC indexing requested
  • ClickUp task: SEO-COL-[CollectionName] moved to SEO_DEPLOYED

Quality Bar

  • Collection description 400+ words, benefit-focused, keyword-natural
  • Schema validates without errors (Google Rich Results Test)
  • CWV all "Good" on mobile (or documented plan to fix)
  • At least 5 internal links established
Workflow 3: Google Shopping Feed Maintenance & Optimization

Trigger

Weekly scheduled review, or alert from Merchant Center (disapprovals, feed errors)

Inputs

  • Current feed export from Shopify
  • Merchant Center diagnostics report
  • Recent product additions/changes from P3-SB-GENERAL

Steps

  1. Feed Health Audit (15-20 min)
    • Check Merchant Center for disapprovals, warnings, pending items
    • Review feed diagnostics: missing attributes, pricing errors, image issues
    • Identify products with low impressions or zero clicks
  2. Critical Issue Resolution (30-45 min)
    • If disapprovals: Fix missing GTINs, incorrect pricing, policy violations
    • If image issues: Replace low-res images, fix broken URLs
    • If description issues: Remove promotional text, ensure compliance
    • Coordinate with P3-SB-GENERAL for Shopify-side fixes
  3. Feed Optimization (30-45 min)
    • Enhance product titles: [Brand] + [Product Type] + [Key Attribute]
    • Add product_type taxonomy (Google Product Categories)
    • Optimize custom labels for campaign segmentation
    • Ensure all variants mapped correctly (size, color)
    • Add promotional overlays if running sale (sale_price field)
  4. PMax Campaign Coordination (15-20 min)
    • Review PMax performance with P3-MB-BUYER
    • Identify top-performing product groups
    • Flag feed improvements that may help campaigns (e.g., better titles)
  5. Documentation (10 min)
    • Export feed diagnostics to Drive (/04_Tracking_and_Feeds/)
    • Log changes in Notion SEO Projects DB
    • Update ClickUp task status

Outputs

  • Feed errors resolved (or escalated)
  • Optimized feed re-uploaded to Merchant Center
  • Documentation in Drive and Notion
  • Coordination notes shared with Media Buyer

Quality Bar

  • Zero critical disapprovals within 48 hours
  • All active products have: title, description, price, image, GTIN/MPN (or exemption)
  • Feed validation passes with no errors in Merchant Center
  • Custom labels used for segmentation (margin tier, seasonality, etc.)
Workflow 4: Core Web Vitals (CWV) Improvement Sprint

Trigger

GSC CWV report shows declining "Good URLs" percentage, or new template/products deployed

Inputs

  • GSC Core Web Vitals report (mobile + desktop)
  • PageSpeed Insights audits for key URLs
  • Lighthouse CI data (if available)

Steps

  1. Prioritization (20-30 min)
    • Identify URLs with "Poor" or "Needs Improvement" ratings
    • Prioritize by traffic/revenue impact (focus on top collections, best-sellers)
    • Group by issue type: LCP, FID, CLS
  2. LCP Optimization (1-2 hours)
    • Optimize hero images: WebP format, responsive sizes, preload critical images
    • Remove render-blocking resources (defer JS, inline critical CSS)
    • Enable CDN caching for images/assets
    • Minimize server response time (TTFB)
  3. FID/INP Optimization (1-2 hours)
    • Defer non-critical JS (analytics, chat widgets)
    • Code-split large JS bundles
    • Remove unused third-party scripts
    • Optimize event handlers (debounce/throttle)
  4. CLS Optimization (30-60 min)
    • Set explicit width/height on images and embeds
    • Reserve space for dynamic content (ads, banners)
    • Avoid inserting content above existing content
    • Use CSS aspect-ratio for responsive elements
  5. Testing & Validation (30-45 min)
    • Re-run PageSpeed Insights on fixed URLs
    • Test on real mobile device (throttled 3G)
    • Monitor GSC for CWV improvements (takes 28 days to fully reflect)

Outputs

  • CWV improvements deployed
  • Before/after PageSpeed scores documented in Drive
  • ClickUp task: SEO-CWV-[Issue] moved to SEO_DEPLOYED
  • Monitoring dashboard updated with new baseline

Quality Bar

  • Target: 75%+ URLs in "Good" category (GSC report)
  • Mobile PageSpeed score > 80 for key pages
  • LCP < 2.5s, FID < 100ms, CLS < 0.1 (75th percentile)
  • No regressions on other metrics (accessibility, best practices)
